# HashLens demo fixtures (fully synthetic)

Visibly fictional identities on `.example` domains — no real people, orgs,
or credentials. Used by the suite demo runbook
(openi-kernel `docs/DEMO_RUNBOOK.md`), HashLens act:

1. Match workspace → paste `discovered-hashes.txt` as the discovered set
   (auto-detected MD5/SHA-256 mix).
2. Paste `candidate-selectors.txt` as candidates — Gmail-style
   normalization produces matches for the two Meridian Freight mailboxes
   and the `anchor2026!` password.
3. Save to a case ("Bluewater Consortium exposure — DEMO"), then export the
   client summary and the case packet from Case results.
